### 🌱 OSCP Journey & Lab Methodologies

I am actively preparing for the **OSCP (Offensive Security Certified Professional)** exam. My lab hours are spent sharpening a rigorous, tactical penetration testing methodology spanning five core phases:

*   **Phase 1: Reconnaissance & Active Scanning** 
    *   Network discovery and fast host identification using `arp-scan`, multi-threaded bash ping sweeps, and precise `nmap` target sweeping.
*   **Phase 2: Protocol Enumeration & Vulnerability Discovery**
    *   **Web Services**: Fuzzing parameters and directory discovery via `gobuster`, `ffuf`, and `wfuzz`; configuration auditing with `nikto`.
    *   **Active Directory & Network Protocol Auditing**: Enumerate domain infrastructure using `kerbrute`, `impacket-GetADUsers`, `impacket-lookupsid`, `smbclient`, `snmpwalk`, and `onesixtyone`.
    *   **Databases**: Manual UNION-based SQL injection mapping and automated triage via `sqlmap`.
*   **Phase 3: Exploitation & Initial Access**
    *   Crafting targeted exploitation paths, executing log poisoning vectors (LFI to RCE via SSH auth logs), manual reverse shell staging, and deploying custom Python or Bash payloads.
*   **Phase 4: Privilege Escalation (Linux & Windows)**
    *   TTY terminal stabilization, exploiting misconfigured SUID binaries, analyzing custom cron jobs for wildcard parameter injection attacks, structural credential harvesting, and auditing system binaries.
*   **Phase 5: Active Directory Compromise & Lateral Movement**
    *   Executing Kerberoasting tactics with `impacket-GetUserSPNs`, extracting DC password hashes using `impacket-secretsdump`, and establishing secure remote management access via `evil-winrm`.
